# 01. Постановка задачи, ограничения, критерии успеха

> Источник: трек «Научный клубок» хакатона «Норникель AI Science Hack» (июль 2026), **итоговое ТЗ (опубликовано 02.07 22:00** — содержательно совпадает с ранним описанием «Единая карта знаний R&D»; сверка проведена, расхождений с планом нет), RFC-001 «R&D Knowledge Map», Q&A организаторов в Telegram.

## 1. Суть задачи (из ТЗ)

Создать **единую карту знаний R&D** для горно-металлургической отрасли: knowledge graph / поисково-аналитическую систему, связывающую:

- научные публикации и отчёты (обзоры, патенты, диссертации);
- экспериментальные данные (протоколы опытов, параметры процессов, результаты анализов);
- технологические решения (схемы циркуляции электролита, конструкции диафрагменных ячеек, подача шихты в ПВП);
- материалы и вещества (сульфаты, хлориды, никелевые катоды, техногенный гипс);
- оборудование и установки (ванны электроэкстракции, печи взвешенной плавки, газоочистка);
- исследовательские команды и экспертов (лаборатории, авторы, носители компетенций);
- выводы и рекомендации (подтверждённые эффекты, ограничения, области применения).

Целевой класс запросов: **«что уже делали по сплавам X при режиме Y и какой был эффект на свойство Z»** + связанные сущности, история решений, пробелы в данных.

## 2. Ключевые требования (нормализованные)

### 2.1. Жёсткие (провал = провал решения)

- **KR-1. Числовая точность.** «Ошибки в извлечении концентраций или температур недопустимы» — единственное требование ТЗ со словом «недопустимо». Извлечение чисел, единиц, операторов (≤, ≥, диапазон, ±, «от», «не более») — с точностью, близкой к 100%.
- **KR-2. Многопараметрические запросы**: материал + процесс + условия + география + временной диапазон в одном запросе.
- **KR-3. Верификация знаний**: источник, уровень достоверности, дата актуализации, статус валидации у каждого факта; provenance до цитаты (span) в документе.
- **KR-4. География**: различение и фильтрация «отечественная практика» / «зарубежная практика».
- **KR-5. Числовые ограничения и диапазоны** в поиске: концентрации, температуры, скорости потока, экономические показатели.
- **KR-6. Масштабируемость доменов**: гидрометаллургия, пирометаллургия, экология, переработка отходов, далее РЗЭ и др.
- **KR-7. Самостоятельное формирование новых связей при расширении базы** (прямой ответ организаторов от 01.07.2026): при добавлении документов система инкрементально линкует сущности, пересчитывает консенсус/противоречия/пробелы без ручного вмешательства.

### 2.2. Функциональные

- Импорт: PDF, DOCX, TXT/MD, CSV/XLSX (каталог экспериментов), справочники материалов/оборудования/единиц, перечень сотрудников/лабораторий, таксономия тегов. RU + EN.
- NLP-извлечение: сущности (12 типов по RFC), связи (17 типов), числовые ограничения и условия.
- Словарь синонимов RU/EN: «электроэкстракция»/«electrowinning», «ПВП»/«печь взвешенной плавки», «МПГ»/«PGM», «сухой остаток»/«TDS».
- Поиск: естественный язык, семантический + точный + графовый + числовой; фильтры (материал/процесс/оборудование/география/год/тип источника/достоверность/диапазоны параметров); сравнительные запросы (RU vs мир, метод A vs B).
- Синтез ответа: краткий вывод, методы, evidence-таблица, числовые параметры, источники, противоречия, пробелы, уверенность, эксперты, подграф.
- Аналитика: консенсус/разногласия, gap-анализ (неизученные комбинации «материал–режим–условие», «только зарубежные данные»), рекомендации смежных кейсов и экспертов.
- Визуализация графа: цепочки «материал → процесс → оборудование → результат», подсветка противоречий и пробелов, эксперты по теме.
- Дашборд руководителя: покрытие по направлениям, активность лабораторий, зоны риска.
- Сравнительные таблицы технологий: эффективность / CAPEX / применимость в холодном климате / экологические ограничения.
- Экспорт: Markdown, PDF, JSON, JSON-LD, CSV.
- Уведомления о новых публикациях по теме (подписки).
- Ручная корректировка графа экспертами с фиксацией автора/даты.
- Хранение графа знаний с поддержкой сложных запросов на обход связей (Cypher/Gremlin **или аналог** — наш аналог: таблица рёбер + рекурсивные CTE, обоснование в [02-hypotheses.md](02-hypotheses.md) E4/E6); онтология ТЗ: Material, Process, Equipment, Property, Experiment, Publication, Expert, **Facility**; версионирование фактов: отслеживание изменений выводов при появлении новых источников.
- RBAC (роли ТЗ): исследователь, аналитик, руководитель проекта, администратор, внешний партнёр; **+ расширение: эксперт-валидатор** (нужен для «ручной корректировки графа экспертами» из доп. пожеланий ТЗ); уровни доступа документов public/internal/confidential/restricted; аудит запросов/просмотров/экспорта.

### 2.3. Нефункциональные

- **Производительность**: сложный запрос с обходом 3–4 уровней связей и числовой фильтрацией ≤3–5 с при 1 млн сущностей; MVP: типовой ответ ≤10 с, поиск по 1–10 тыс. чанков ≤5 с, загрузка документа ≤30 с.
- **Надёжность**: устойчивость к битым документам, повторный запуск ingestion, не терять старые версии фактов, детальное логирование, мониторинг.
- **Расширяемость**: модульная архитектура; новые источники (в т.ч. датчики установок), новые типы сущностей (например, «экономический показатель»), новые домены.
- **Мультиязычность**: RU/EN документы и запросы, кросс-языковое сопоставление терминов.
- **Интуитивный интерфейс**: исследователь без знания графовых БД формулирует запрос на естественном языке.

## 4. Персоны и их «job to be done»

1. **Исследователь** — «за минуты понять, что уже известно по моей комбинации материал×процесс×условия, с числами и первоисточниками».
2. **Аналитик** — «собрать evidence-таблицу и сравнение технологий для отчёта, выгрузить в MD/PDF».
3. **Руководитель R&D** — «увидеть, где знание тонкое/противоречивое/только зарубежное, чтобы планировать портфель исследований».
4. **Эксперт-валидатор** — «подтвердить/отклонить извлечённые факты, поправить связи, оставить след».
5. **Администратор** — «источники, справочники, доступы, ошибки ingestion».

## 5. Критерии успеха нашего решения

### Продуктовые (демо перед жюри)
- 6 эталонных запросов Q1–Q6 отрабатывают стабильно, с evidence и provenance до цитаты.
- Ответ «дни → минуты»: время сборки мини-обзора ≤1 мин против ручных дней.
- Три «вау»-дифференциатора от plain RAG: (а) числа с гарантией источника, (б) верифицированные противоречия с причиной, (в) heatmap пробелов + эксперты.

### Метрики качества (gold-set, см. [13-evaluation.md](13-evaluation.md))
- `numeric_extraction_precision ≥ 0.98`, `unit_normalization_accuracy ≥ 0.99` (детерминированное ядро);
- `hallucinated_numbers_rate = 0` (numeric guard, архитектурная гарантия);
- `retrieval precision@10 ≥ 0.80`, `MRR ≥ 0.75`;
- `contradiction_precision ≥ 0.80` (на judge-подтверждённых);
- `expert_discovery top-3 hit-rate ≥ 0.90`;
- `latency p95 ≤ 5 c` (полный ответ), first token ≤ 1.5 с.

### Инженерные (прод-уровень)
- Горизонтальное масштабирование stateless-сервисов; деградация по уровням (LLM недоступен → экстрактивные ответы; embeddings недоступны → FTS+SQL).
- Переиндексация корпуса без даунтайма (replay событий).
- RPO ≤ 15 мин, RTO ≤ 30 мин (см. [11-nfr.md](11-nfr.md)).

## 6. Что сознательно НЕ делаем (анти-скоуп MVP)

- Полная интеграция с корпоративными IAM/DMS заказчика (готовим интерфейсы: OIDC, S3-совместимое хранилище; сама ролевая модель и уровни доступа — в скоупе, см. §2.2).
- Полная онтология отрасли — только ядро доменов датасета + механизм расширения справочников.
- Автопринятие технологических решений — система даёт evidence, решает человек.
- OCR сканов и извлечение из изображений — за скоупом MVP (текстовый слой гарантирован организаторами), архитектурно поддержано флагом парсера.
- Real-time обработка миллионов документов — целевой объём: 10⁴–10⁵ документов, 10⁶ сущностей, 10⁷ фактов.
