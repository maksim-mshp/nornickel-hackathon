# 17. Фронтенд: дизайн-система, страницы, UX

Требование: оригинальный, удобный интерфейс с характером, а не «типовой AI-дашборд». Документ следует принципам [Anthropic Frontend Aesthetics](https://platform.claude.com/cookbook/coding-prompting-for-frontend-aesthetics): явный выбор типографики (никаких Inter/Roboto/system), целостная тема на CSS-переменных с доминантой и резкими акцентами, культурно-контекстное вдохновение, motion в высокоударных моментах, многослойные фоны вместо плоских заливок.

## 1. Дизайн-концепция: «Полярная ночь / Электролит»

Продукт — аналитический workspace для металлургов Заполярья. Визуальный язык собран из **реального контекста домена**, а не из трендов:

- **Полярная ночь** — глубокий сине-графитовый фон (не чёрный), длинные тени, холодный воздух. Тёмная тема — основная: исследователь работает с плотными данными часами.
- **Электролит** — изумрудно-бирюзовый растворов сульфата никеля (характерный цвет цеха электроэкстракции) — главный акцент: действия, подтверждённость, ссылки.
- **Расплав** — раскалённый медно-оранжевый — только для конфликтов знаний (противоречия, риск). Тепло = трение.
- **Инженерная документация** — миллиметровая сетка, штампы чертёжной рамки (ЕСКД), штриховка «нет данных», монолинейные изолинии месторождений в фоновых паттернах. Provenance выглядит как печать ОТК, а не как сноска.
- **Числа — первоклассные граждане** (ядро решения): все значения, единицы, операторы — моноширинно, с табличными цифрами; guard-подтверждение — визуальная «пломба».

Светлая тема — «Протокол»: бумага слоновой кости, тёмно-синяя тушь, те же акценты глуше. Переключается, default — ночь.

### 1.1. Типографика (кириллица обязательна — это отсекло Clash Display/Satoshi/Bricolage)

Заявка выбора (по методике cookbook): **Unbounded + Golos Text + JetBrains Mono** — все с полноценной кириллицей, все на Google Fonts (self-host через `next/font`, без внешних запросов в проде).

| Роль | Шрифт | Начертания | Где |
|---|---|---|---|
| Display | **Unbounded** (широкий, инженерно-брутальный, вариативный) | 700–900 | заголовки экранов, KPI на дашборде, цифры-«приборы»; дозированно |
| Текст/UI | **Golos Text** (гуманистический гротеск, родная кириллица — НЕ Inter) | 400 / 500 / 600 | всё остальное |
| Данные | **JetBrains Mono** (tabular figures) | 400 / 700 | числа, единицы, операторы, цитаты, слаги, код |

Правила: контраст пар — экстремальный (Unbounded 900 рядом с Golos 400); скачки размеров ≥3× — шкала `13 → 14 → 20 → 40 → 64`; операторы `≤ ≥ ± —` в фактах — акцентным цветом; никаких «полужирный 600 на 16px» как единственной иерархии.

### 1.2. Цвет (CSS-переменные, доминанта + резкие акценты)

```css
:root[data-theme="night"] {           /* основная: полярная ночь */
  --bg-0:#0B1014; --bg-1:#111820; --bg-2:#16202B;   /* слои глубины */
  --grid:rgba(126,166,192,.05);                      /* миллиметровка */
  --ink-0:#E8EDF2; --ink-1:#9FB1BF; --ink-2:#5E7280;
  --electrolyte:#2EC79E; --electrolyte-deep:#0F5D49; /* акцент: действия/подтверждено */
  --melt:#FF6A3D; --melt-deep:#8A2E14;               /* противоречия/риск */
  --anode:#D9A441;                                   /* экспертная печать, валидация */
  --void:#7A8CA6;                                    /* пробелы (со штриховкой, не заливкой) */
  --focus:#66E2C2;
}
:root[data-theme="protocol"] {        /* светлая: бумага протокола */
  --bg-0:#F3EFE6; --bg-1:#ECE6D9; --bg-2:#FFFFFF;
  --grid:rgba(27,42,56,.06);
  --ink-0:#1B2A38; --ink-1:#44586A; --ink-2:#7C8B98;
  --electrolyte:#127A5F; --melt:#C2431F; --anode:#8F6A1E; --void:#5E7189;
}
```

Семантика жёсткая: электролит = достоверно/действие; расплав = конфликт; латунь = эксперт/печать; void+штриховка = отсутствие данных. **Запрещено:** пурпурные градиенты, «стекло» с blur на всём, радуга статусов.

### 1.3. Фоны и глубина

- Базовый слой: `--bg-0` + едва заметная миллиметровая сетка 8px (CSS `background-image: linear-gradient×2`), под контентом — тише (маска).
- За ключевой панелью ответа — одно большое радиальное свечение `--electrolyte` (3–4% альфы) — «свет цеха в ночи».
- Шапки дашборда/пустые состояния — SVG-паттерн изолиний (топография месторождения), монолинейный, 4% альфы.
- Никаких стоковых иллюстраций и 3D-глянца.

### 1.4. Motion (высокоударные моменты, CSS-first + Motion для React)

- **Один оркестрированный вход** на экран: панели поднимаются stagger'ом 40 мс (max 8 элементов, `animation-delay`), длительности 240–400 мс, easing `cubic-bezier(.2,.8,.2,1)`.
- **Стрим ответа** — главный спектакль: plan-chip «защёлкивается» → строки evidence приезжают каскадом по мере SSE → числа при прохождении guard получают штамп-пломбу (масштаб 1.15→1, 180 мс) — момент доверия, ради него всё.
- Ego-граф: плавная стабилизация физики 600 мс; рёбра CONTRADICTS — медленная пульсация свечения (2 с цикл).
- Микроинтеракции — скупо: hover карточек = подъём тени + сдвиг 1px, не зоопарк эффектов. `prefers-reduced-motion` отключает всё, кроме opacity.

### 1.5. Фирменные компоненты (узнаваемость продукта)

1. **Штамп-провенанс** — рамка в стиле углового штампа чертежа: `№ документа · стр. · год · метод извлечения`; клик — цитата с подсветкой span. Есть у каждого факта. JetBrains Mono, тонкая двойная рамка.
2. **Guard-пломба** — лента «☑ 14/14 чисел сверены с источниками» с перфорацией по краю; при деградации — жёлтая «экстрактивный режим».
3. **Консенсус-спектр** — кастомный SVG: горизонтальная ось параметра (в SI), каждый источник — штрих-интервал своим диапазоном, зона пересечения залита электролитом, agreed_range — маркер. Виден конфликт диапазонов глазами (killer-виз для Q2).
4. **Карточка противоречия** — два факта колонками, между ними «линия разлома» цвета расплава; внизу плашка судьи: конфаундеры + уверенность; кнопки эксперта (подтвердить/отклонить/разрешить).
5. **Ячейка heatmap** — заливка score градиентом bg→electrolyte; «нет экспериментов» — инженерная штриховка 45°; бейджи причин (`только зарубежные`, `устарело`) — микротеги.
6. **Число-факт** — атом всего UI: `0.6–0.9 м/с` = значение (Mono 700, ink-0) + оператор (акцент) + единица (Mono 400, ink-1); всегда одинаково, везде.

## 2. Стек

| Слой | Выбор | Примечание |
|---|---|---|
| Framework | Next.js 15 (App Router, standalone) + TypeScript strict | SSR только для shell; данные — клиентские |
| Стили | Tailwind CSS v4 + токены из §1.2 (CSS vars) | никакого инлайн-хардкода цветов |
| Примитивы | shadcn/ui (Radix) — перекрашены токенами | диалоги, поповеры, табы: доступность бесплатно |

| Граф | Cytoscape.js (+cose-bilkent layout) | ego-граф, стили из токенов |
| Кастом-визы | SVG руками (консенсус-спектр, heatmap, изолинии) | не тащим тяжёлые чарт-либы |
| Motion | CSS + Motion (framer-motion) точечно | §1.4 |
| Шрифты | `next/font` self-host: Unbounded, Golos Text, JetBrains Mono | нет внешних CDN в проде |

| Язык интерфейса | только русский (next-intl, словарь `ru`; переключатель языка и EN-локаль удалены) | мультиязычность RU/EN — на уровне корпуса и поиска (bge-m3, FTS ru/en, canonical RU/EN), не в UI-чрому |

Структура: `ui/src/{app,widgets,features,entities,shared}` (FSD-подход): `entities/fact` (число-факт, штамп), `features/ask` (SSE-лента), `widgets/evidence-table`, `widgets/consensus-spectrum`, `widgets/ego-graph`, `shared/ui` (обёртки shadcn), `shared/api` (генерированный клиент).


- **⌘K — командная палитра** (единственный вход всюду): спросить вопрос, перейти к сущности/документу/эксперту, применить пресет, сменить тему. Поиск сущностей — с типом и слагом.
- Роли режут навигацию (RBAC, ADR-6): Researcher — Поиск/Эксперименты/Эксперты; Manager + Покрытие; Expert + Ревью; Admin + Документы/Словари; External Partner — только Поиск по разрешённым данным. В шапке — блок пользователя (инициалы в рамке-штампе, роль, выход); в demo-режиме — переключатель ролей (удобно показать RBAC жюри).
- Desktop-first (аналитический инструмент): ≥1440 полный, 1100–1440 инспектор поверх (overlay), <1100 фильтры в drawer; мобильный — только чтение сохранённых ответов.

## 4. Страницы

### 4.1. `/` — Research Workspace (главная, 80% времени пользователя)

**Стартовый экран (idle) — минималистичный лендинг в стиле поисковика** (решение владельца): по центру только вордмарк `◆ kmap` и одно крупное поле поиска (плюс компактная строка пресетов-пилюль Q1–Q6 как ярлыки), без hero-текста, «что нового» и сетки карточек — «одно поле поиска, без лишних элементов». После первого запроса экран разворачивается в три зоны: **Паспорт запроса** (264px) · **Лента ответа** (fluid, max 880px) · **Инспектор** (360px).

```
┌ПАСПОРТ─────────┐┌ЛЕНТА ОТВЕТА──────────────────────────┐┌ИНСПЕКТОР────────┐
│ материал    ▾  ││ ┌ plan-chip ───────────────────────┐ ││ (пусто → граф)  │
│ процесс     ▾  ││ │ intent: подбор технологии    ✎   │ ││ ┌─────────────┐ │
│ география ru/w ││ │ [сульфаты 200–300 мг/л][TDS≤1000]│ ││ │ ЦИТАТА       │ │
│ год 2018──2026 ││ └──────────────────────────────────┘ ││ │ «скорость…»  │ │
│ параметр+опер. ││ ▍Summary  ▍Evidence 12  ▍Противо- 2 │ ││ │ ─────────── │ │
│ [+ ограничение]││ ▍Пробелы 3 ▍Эксперты 5   (вкладки)  │ ││ │ ШТАМП №017  │ │
│ достоверн. ●── ││                                      │ ││ │ стр.12 2023 │ │
│ ─────────────  ││ ⛿ guard: 14/14 чисел сверены        │ ││ └─────────────┘ │
│ сохранённые    ││ [экспорт MD/CSV] [поделиться]        │ ││ ▸ ego-граф      │
└────────────────┘└──────────────────────────────────────┘└─────────────────┘
```

- **Паспорт запроса** — фильтры как «поля протокола»: сущности-чипы с автодополнением (canonical + синоним, который ввёл пользователь, показывается серым), числовое ограничение — конструктор `параметр · оператор · значение · единица` (единицы из реестра, конверсия в SI подсказкой). Фильтры двусторонни с plan-chip: правка чипа обновляет паспорт и наоборот.
- **Plan-chip** («как вас поняли») — приходит первым SSE-событием: intent + распознанные сущности/ограничения как редактируемые чипы; парсер-источник (LLM/rules) видно по микроиконке. Ошибся разбор — правишь чип, перезапуск без повторного набора.
- **Summary** — стримится; каждое число — компонент «число-факт» со ссылкой [Fn]; по завершении guard-пломба защёлкивается (§1.4).
- **Evidence** — таблица: Источник (штамп) · Метод/Subject · Параметр (число-факт) · Условия (микротеги) · География · Год · Score (мини-бар с разбивкой по hover). Строки виртуализированы; `j/k` — навигация, `o` — источник в инспектор, `Enter` — цитата.
- **Противоречия** — карточки §1.5.4 + отдельная секция «различия контекста» (не конфликты).
- **Пробелы** — ячейки с причинами + «смежные области» как кликабельные под-запросы.
- **Эксперты** — карточки: ФИО, лаборатория, вес-бар, evidence-цепочка (док→эксперимент→год), кнопка «показать в графе».
- **Инспектор** — контекст клика: цитата с подсветкой span в тексте чанка, штамп, соседние факты документа; вкладка «Граф» — ego-граф вокруг сущностей плана (Cytoscape, легенда типов, red-pulse CONTRADICTS).
- **Пустое состояние** — не пустота, а «журнал»: 6 пресетов Q1–Q6 как каталожные карточки со штампами + последние запросы команды + строка «что нового в базе» (из `epistemic.updated`).
- Ошибки: LLM недоступен → жёлтая лента «экстрактивный режим» (факты есть, текста нет — честно); поиск пуст → «расширить: убрать фильтр X? смежные ячейки?».

### 4.2. `/entity/[slug]` — Паспорт сущности

Шапка: тип-бейдж, canonical RU/EN, синонимы (чипы, серые — pending), счётчики (документы/факты/эксперименты/эксперты) в Unbounded. Ниже — три колонки: **Параметры** (список параметров с консенсус-спектрами §1.5.3 — сразу видно изученные диапазоны и конфликты), **Связи** (сгруппированные рёбра: материалы/оборудование/процессы, вес = толщина линии-коннектора), **Люди** (эксперты с весами). Вкладка «Хронология» — факты по годам (timeline с точками-источниками, видно устаревание). Кнопка «Спросить об этом» — префилл workspace.

### 4.3. `/experiments` — Каталог экспериментов

Плотная таблица (13px, зебра тонкими линиями сетки): ID · материал · процесс · условия (число-факты) · результат · источник-штамп · confidence. Слева фасеты; сверху «сравнить выбранные» → режим сравнения: строки становятся колонками, параметры — строками, различия подсвечены (максимум 4). Экспорт CSV. Пустая ячейка условия — штриховка, не прочерк (единый язык «нет данных»).

### 4.4. `/coverage` — Карта покрытия (руководитель)

- Верх: 4 KPI-«прибора» (Unbounded 64): документы / подтверждённые факты / противоречия / пробелы; под каждым — микротренд.
- Центр: **heatmap материал×процесс** (§1.5.5): оси переключаемы (домен/условие), клик по ячейке — боковой drawer: счётчики, причины гэпа, топ-документы, эксперты соседних ячеек, кнопка «сформировать запрос».
- Низ: «Зоны риска» — три списка: противоречивые кластеры (расплав), устаревшие темы (>7 лет), «только зарубежное» (флаг-иконка). Каждая строка — переход в workspace с готовым запросом.
- Фон шапки — изолинии (§1.3). Экспорт: PNG heatmap + CSV.

### 4.5. `/experts` — Институциональная память

Поиск по теме/сущности → карточки экспертов: аватар-инициалы в рамке-штампе, лаборатория, вес-бар по теме, спарклайн активности по годам, evidence-цепочка (раскрывается: документы → эксперименты). Фильтр «только активные (≤3 лет)». Режим «карта»: биграф человек—тема (Cytoscape). Приватность: роль external_partner видит лаборатории без ФИО.

### 4.6. `/documents` — Реестр и конвейер (роли admin/expert)

Таблица документов: название · тип · язык · география · access-бейдж · статус конвейера (шаги-точки: registered→parsed→extracted→indexed, ошибка — расплав с причиной по hover) · счётчики фактов. Dropzone сверху (drag-n-drop, множественная, манифест). Ряд раскрывается в «паспорт документа»: версии, метрики качества экстракции (nc_suspect_rate, llm_valid_rate), кнопка reindex.

### 4.7. `/review` — Очередь ревью (роль expert)

Три стопки-вкладки со счётчиками: **Сущности pending** (merge/approve с показом similarity-кандидатов), **Числа-сироты** (orphan: цитата + кандидаты привязки), **Противоречия suspected** (карточка §1.5.4 + вердикт судьи → подтвердить/отклонить/разрешить с комментарием). Работа с клавиатуры: `a` approve, `r` reject, `→` следующий. Каждое действие — тост с undo (5 с) и запись в fact_history.

### 4.8. `/dictionaries` — Словари (роль admin)

Две панели: синонимы (canonical ← алиасы с языком/источником/статусом; inline-добавление) и единицы (код, написания, размерность, SI-фактор; тест-строка «распарсить пример» — живой прогон numcore-правила). Версии правок с автором.

### 4.9. `/answers/[id]` — Сохранённый ответ

Read-only слепок AnswerDoc: план, summary, evidence, пломба guard, штампы. Печатная версия = светлая тема «Протокол» автоматически (`@media print`). Это же — формат share-ссылки для жюри.

### 4.10. `/login`

Минимум: логотип-глиф, одна кнопка OIDC (Keycloak) — либо выбор demo-роли (`auth.mode: demo`); фон — изолинии + свечение. Никаких маркетинговых слоганов.

## 5. Доступность и качество

- Контраст AA на обеих темах (проверка в CI: axe-core на Storybook-стенде ключевых компонентов).
- Всё с клавиатуры; фокус-кольцо `--focus` 2px; skip-link; таблицы с заголовками; aria-live="polite" на SSE-ленте (screenreader получает summary после guard, не поток дельт).
- `prefers-reduced-motion`, `prefers-color-scheme` (первичная тема), сохранение в localStorage.
- Производительность: initial JS ≤ 250 КБ gz (Cytoscape и Motion — lazy по маршруту/видимости); LCP ≤ 1.5 с на demo-VM; шрифты subset (кириллица+латиница+знаки ≤±≥·°), `font-display: swap`; виртуализация всех длинных списков.
- Скелетоны — контурные (линии сетки), не серые «кости»; тосты — нижний левый, с undo где применимо.

## 6. Анти-чеклист (запрещено — «AI slop»)

- Inter/Roboto/Arial/system-ui; Space Grotesk «по привычке».
- Пурпурно-фиолетовые градиенты, градиент-кнопки, glassmorphism поверх всего.
- Одинаковые карточки 12px radius с тенью `0 4px 12px rgba(0,0,0,.1)` на белом.
- Эмодзи в заголовках интерфейса; иконки-стикеры; конфетти.
- Три колонки «фич» с иконкой-заголовком-текстом; hero с центрированным слоганом.
- Спиннер на весь экран (только скелетоны/прогресс конвейера); «карусели».
- Цвета в компонентах мимо токенов; произвольные оттенки статусов.

## 7. Визуальная верификация (обязательный цикл для агентов)

UI не пишется вслепую. После каждой значимой правки — цикл через **chrome-devtools MCP** (доступен агентам; любые CLI-команды тоже разрешены):

1. Поднять dev-сервер (`task dev` / `pnpm dev`), открыть изменённую страницу в браузере.
2. **Сделать скриншот и посмотреть на него** — сверить с макетом раздела §4, токенами §1.2 и анти-чеклистом §6 (шрифты применились? числа в Mono? штампы/пломбы на месте? нет «AI slop»-паттернов?).
3. Проверить консоль браузера — ноль ошибок/варнингов React и CSP.
4. Прогнать обе темы (night/protocol) и узкий вьюпорт (1100px — инспектор в overlay).
5. Для стрима — скриншот в середине SSE-ответа (скелетоны/каскад строк) и после guard (пломба защёлкнута).

Скриншоты промежуточных состояний складывать в `.tmp/ui-shots/` (не коммитятся). Правка не считается завершённой, пока агент не увидел результат глазами.
