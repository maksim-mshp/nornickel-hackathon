# .agents — проектная документация R&D Knowledge Map (kmap)

Пакет документов системного анализа для трека «Научный клубок» (Норникель AI Science Hack 2026). Здесь — **вся** проектная истина: требования, выбор решения, архитектура, модель данных, контракты, план. Код обязан соответствовать этим документам; расхождение — повод обновить документ осознанно (через PR), а не игнорировать.

## Инварианты проекта (нарушать нельзя)

1. Числа/единицы/операторы извлекаются и проверяются **только детерминированным кодом** (numcore); LLM не может породить или изменить число (guard).
2. Каждый факт — с provenance до цитаты и span; факты не удаляются, только supersede/статус.
3. Один стор — PostgreSQL (+pgvector); новые хранилища — только через пороги пересмотра ([03](03-architecture.md) §7).
4. Все контракты — в `contracts/`; изменения только аддитивные (buf breaking check, oasdiff).
5. Временные файлы — только в `.tmp/`.
6. Недостающие ресурсы/доступы — запрашивать у владельца ([15](15-resources.md)), не хардкодить обходы.
